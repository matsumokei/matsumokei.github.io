長らく別のツールでブログを書いていましたが、Astro + Markdown 構成に移行しました。

理由はシンプルで、記事を書くこと以外に気を取られたくなかったからです。テーマの見た目を調整したり、CMS の管理画面と格闘したりする時間を減らし、`src/content/blog/` に Markdown ファイルを置くだけで公開できる状態を目指しました。

## この構成でできること

- Markdown / MDX で記事を書く
- Astro の Content Collections で型安全にフロントマターを扱う
- サーバーサイドの JS を出荷せず、静的な HTML として配信する

今後、技術的なメモや読んだ本の記録などをここに残していく予定です。
